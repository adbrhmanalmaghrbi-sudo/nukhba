# المهمة التالية - مشروع Nukhba

## آخر تحديث: 2026-07-21

## الأولوية الحالية
تشغيل flutter test للتحقق من ان توليد الكود لم يكسر اي منطق عمل موجود قبل الانتقال لاعداد Supabase

## خطوات التنفيذ المقترحة
1. تشغيل: cd /workspaces/nukhba && flutter test 2>&1 | tail -50
2. مراجعة اي اختبارات فاشلة بشكل منهجي حسب الميزة
3. بعد نجاح الاختبارات الانتقال لمرحلة الاتصال بـ Supabase
4. التحقق من .env.example وانشاء .env فعلي بمفاتيح Supabase الحقيقية
5. بعد التحقق من Supabase مناقشة استراتيجية النشر

## ملاحظات مهمة للاستمرارية
- لا تعد توليد g.dart مرة اخرى ما لم تتغير تعريفات riverpod او freezed
- عند اضافة ميزة جديدة تستخدم riverpod يجب اعادة تشغيل build_runner
- بنية المشروع monorepo بـ Clean Architecture
